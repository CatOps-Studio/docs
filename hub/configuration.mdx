CatOps Hub is designed to work out of the box with minimal configuration.

## Port Configuration

### During Installation

```bash
curl -fsSL https://get.catops.app/hub.sh | sh -s -- -p 9000
```

### After Installation

Edit the systemd service:

```bash
sudo systemctl edit catops-hub
```

Add:
```ini
[Service]
ExecStart=
ExecStart=/opt/catops/catops-hub serve --http 0.0.0.0:9000
```

Then restart:
```bash
sudo systemctl daemon-reload
sudo systemctl restart catops-hub
```

## Auto-Update

### Enable During Installation

```bash
curl -fsSL https://get.catops.app/hub.sh | sh -s -- --auto-update
```

This creates a systemd timer that checks for updates daily at a random time.

### Check Timer Status

```bash
sudo systemctl status catops-hub-update.timer
```

### Disable Auto-Update

```bash
sudo systemctl disable catops-hub-update.timer
sudo systemctl stop catops-hub-update.timer
```

## Reverse Proxy

### nginx

```nginx
server {
    listen 80;
    server_name monitoring.example.com;

    location / {
        proxy_pass http://127.0.0.1:8090;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

<Info>
  The `Upgrade` and `Connection` headers are required for Server-Sent Events (SSE) used by real-time metrics.
</Info>

### Caddy

```
monitoring.example.com {
    reverse_proxy localhost:8090
}
```

### With SSL (nginx + certbot)

```bash
# Install certbot
sudo apt install certbot python3-certbot-nginx

# Get certificate
sudo certbot --nginx -d monitoring.example.com
```

## Docker Socket Access

For container monitoring, Hub needs access to the Docker socket.

The installation script adds the `catops` user to the `docker` group automatically.

### Manual Setup

If containers aren't showing:

```bash
# Check if catops is in docker group
groups catops

# Add to docker group if not
sudo usermod -aG docker catops

# Restart Hub
sudo systemctl restart catops-hub
```

### Docker Socket Location

By default, Hub looks for the Docker socket at `/var/run/docker.sock`.

## Data Storage

| Path | Description |
|------|-------------|
| `/opt/catops/catops-hub` | Binary executable |
| `/opt/catops/pb_data/` | PocketBase database |
| `/home/catops/.catops/` | License file |

### Backup

To backup your Hub data:

```bash
sudo cp -r /opt/catops/pb_data ~/catops-backup/
```

### Restore

```bash
sudo systemctl stop catops-hub
sudo cp -r ~/catops-backup/pb_data /opt/catops/
sudo chown -R catops:catops /opt/catops/pb_data
sudo systemctl start catops-hub
```

## Resource Limits

To limit Hub's resource usage, edit the systemd service:

```bash
sudo systemctl edit catops-hub
```

```ini
[Service]
MemoryMax=256M
CPUQuota=50%
```

<Note>
  Hub typically uses very little resources (~30MB RAM, minimal CPU).
</Note>
